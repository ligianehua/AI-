# AI 售后助手

基于大语言模型与多轮对话技术的企业售后服务智能助手，覆盖**客户咨询应答、故障诊断与排障、工单管理、退换货处理、满意度管理、售后数据分析**六大环节，并具备**从客户对话中自我学习提升**的闭环能力，帮助售后团队从"被动响应"转向"主动服务"。

## 快速开始

```bash
# 1. 安装依赖（Python 3.10+）
pip install -r requirements.txt

# 2. 启动（首次运行自动建库并写入演示数据）
python run.py
```

启动后访问：

| 入口 | 地址 | 用途 |
|---|---|---|
| 客户聊天页 | http://127.0.0.1:8000/ | 客户手机号登录，与 AI/人工客服对话 |
| 管理控制台 | http://127.0.0.1:8000/admin | 售后团队：仪表盘/工作台/工单/退换货/知识库/学习审核/主动提醒 |

> 端口 8000 被占用时会自动改用空闲端口，以启动日志打印的地址为准。

**登录账号**（登录页有点击填入的演示提示）：

| 端 | 账号 | 说明 |
|---|---|---|
| 客户端 | 手机号 13800000001~0008 | 演示客户（张伟/李娜/…）；新手机号+姓名自动注册 |
| 管理端 | `admin` / `admin123` | 管理员（全部功能） |
| 管理端 | `agent1` / `agent123` | 客服（工作台/工单/退换货/提醒） |
| 管理端 | `ops1` / `ops123` | 知识运营（知识库/手册/学习审核） |

**无需任何 API Key 即可完整演示**：未配置 Key 时自动进入演示模式（MOCK），由内置规则引擎模拟 AI 决策——工具调用、数据库、工单/退换货状态机、SSE 流式全部走真实代码路径，只有"模型大脑"是模拟的。

## 接入真实大模型（两种方式，二选一）

**方式一：Anthropic 官方 API**（优先级最高）

```bash
# .env 中填入
ANTHROPIC_API_KEY=sk-ant-...
```

使用官方 `anthropic` SDK（默认 claude-opus-4-8）：adaptive thinking、多轮 tool use 循环、SSE 流式、系统提示词+工具定义提示缓存（`cache_control`）、学习分析用结构化输出（JSON Schema）。

**方式二：OpenAI 兼容接口**（API 中转站 / 国产模型：Qwen、GLM、DeepSeek 等）

```bash
# .env 中填入
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.silra.cn/v1
MODEL=qwen3-max
```

通过 `openai` SDK 的 chat.completions 接口对接，自动完成 Anthropic 风格消息 ↔ OpenAI 格式（function calling / role=tool）的双向转换，Agent 引擎零改动。**所选模型必须支持工具调用**——silra.cn 中转站实测可用：`qwen3-max`、`glm-5`（`kimi-k2.6`/`deepseek-chat` 经该中转不返回工具调用，不要选用）。

两种方式都无需改代码，填好 `.env` 重启即生效；全部留空则回到演示模式。

## 功能总览

### 客户端（聊天页）
- **多轮对话**：流式打字机输出，工具调用过程内联展示（"🔍 正在查询知识库…✅"）
- **咨询应答**：先检索知识库再回答（jieba 分词 + SQLite FTS5 中文全文检索，LIKE 兜底）
- **故障排查**：按故障排查决策树逐步引导，一次一个步骤，结论可直通保修维修
- **退换货办理**：查订单 → 核对政策（签收日起 7 天退 / 15 天换 / 保修期修）→ 客户确认 → 提交申请，全程政策校验
- **工单/售后查询**：单号查询进度与流水
- **转人工**：自动创建高优先级工单并附 AI 交接摘要
- **满意度**：对话中口头评分 + 结束会话星级弹窗

### 登录与权限
- 客户端：手机号登录（未注册自动注册），只能访问自己的会话/订单/售后/通知
- 管理端：员工账号 + 三种角色（管理员/客服/知识运营），导航与 API 双重角色校验（越权返回 403）
- token 鉴权（PBKDF2 密码哈希），登出即失效

### 人工客服工作台（人机协作）
- AI 转人工后会话进入**待接入队列**，客服一键接管 → 会话切换人工模式（AI 停止应答，客户端显示"人工客服服务中"横幅）
- 客服与客户**实时对话**（2.5s 轮询），系统提示条展示接入/退出/结束事件
- **AI 推荐话术**：一键让大模型基于对话上下文生成可编辑的回复建议（实测能自动引用上下文中的工单号）
- 处理完可**交还 AI** 继续服务或直接结束会话

### 向量语义检索（混合召回）
- 知识库与手册检索升级为 **FTS5 关键词 + text-embedding-v4 语义向量** 双路召回，RRF 融合排序
- 客户说"扫完不回家"能命中"不回充"、"很久不用怎么收纳"能命中"长期存放"——关键词检索做不到的语义匹配
- embeddings 接口不可用时自动降级为纯 FTS，不影响运行；「重建索引」会补齐全部缺失向量

### 主动提醒引擎（被动→主动）
- 定时扫描（每 30 分钟 + 启动时 + 手动触发）：**保修到期前 30 天提醒**、**按产品类目的保养周期提醒**（滤芯/刷头/电池等）
- **退换货状态变化即时推送**客户通知（审批/收货/完成每一步）
- **差评自动回访**：≤2 星评价自动生成高优先级回访工单 + 内部任务
- 客户端顶栏 🔔 通知中心（未读角标），管理端「主动提醒」页统一跟进

### 拍照报修（图片识别）
- 聊天输入框 📷 上传故障照片/错误截图（JPG/PNG/WebP，≤5MB），视觉模型（qwen-vl-max）自动读图诊断（可识别屏幕错误代码），结果注入 Agent 上下文继续排查流程
- 人工模式下照片直接转给客服；工作台与会话回放均可查看图片

### 扫描版 PDF OCR
- 上传的 PDF 若无文字层（扫描件/纯图片），自动用 pymupdf 逐页渲染 → 视觉模型 OCR 提取文字再切块入库（最多处理前 20 页）

### 质量与运营闭环
- **AI 会话质检**：会话记录页一键质检，LLM 按 服务态度/专业准确/解决效率 三维度打分（0-100）并指出问题与改进建议，分数同列表展示、可导出
- **答案反馈**：客户对 AI 回答 👍/👎；点踩自动把对应问题回流为知识缺口候选（接入自我学习闭环），回放页可见客户反馈标记
- **学习分析自动化**：每小时检查，距上次分析超过 `AUTO_ANALYZE_HOURS`（默认6小时）且有新会话即自动运行，无需手动点击
- **热点预警**：某问题标签近 7 天出现 ≥3 次且未下降，自动生成运营内部任务提醒
- **数据导出**：工单/退换货/会话（含质检分、满意度）一键导出 CSV，Excel 直接打开

### 多渠道接入（OpenAPI）
外部系统（微信公众号网关、小程序服务端、APP）通过同步接口接入同一个 AI 客服：

```
POST /api/channel/message
Header: X-Channel-Key: <CHANNEL_API_KEY>
Body: {"external_user_id": "微信openid", "name": "昵称", "message": "客户消息"}
→ {"conversation_id": 1, "mode": "ai", "reply": "AI回复全文", "tools": [...], "cards": [...]}
```

外部用户自动建档并复用会话；会话被人工接管时返回 `mode: "human"`（消息已转达）。`.env` 配置 `CHANNEL_API_KEY` 启用。

### 管理端（控制台）
- **数据分析仪表盘**：今日会话/AI 自助解决率/转人工率/未结工单/满意度指标卡 + 趋势折线/分类饼图/热点问题条形图/满意度仪表盘
- **工单看板**：5 列 Kanban（待处理→处理中→待客户确认→已解决→已关闭），操作按钮由状态机生成，非法迁移返回 409
- **退换货管理**：RMA 审批全流程（已提交→已批准→待寄回→已收货→处理中→已完成）
- **知识库管理**：增删改查 + 检索调试（实时看 bm25 命中与得分）+ 索引重建，AI 学习条目带徽标
- **产品手册投喂**：上传 PDF / Word / TXT / Markdown 手册 → 自动解析、按章节切块（约500字/块）、进全文索引 → AI 回答客户问题时自动检索手册原文并**标注出处**（手册名+章节）；支持"消化为 FAQ 候选"——LLM 通读手册提炼常见问答，进学习审核队列人工把关后入库
- **学习审核**：一键分析历史会话 → 审核候选（可编辑）→ 入库
- **会话记录**：全量会话回放（含工具调用轨迹）

### 自我学习闭环
1. 离线分析已关闭会话（LLM 结构化输出 / Mock 规则），产出：新问答候选、知识缺口、热点标签
2. 相似候选自动合并计数（jieba 分词 Jaccard 去重），出现 ≥3 次自动标记 🔥热点 并置顶
3. 未解决/转人工/差评会话自动回流为知识缺口
4. **人工审核把关**：编辑后入库知识库（source=learned）→ 下一次客户提问立即命中
5. 热点标签聚合进仪表盘，支撑"主动服务"决策

## 推荐演示脚本

1. 聊天页选择客户「张伟」→ 新会话 → 点「查询我的订单」
2. 输入"我要退货，订单号 SO…-0001"→ 看政策核实 → 回复"确认" → 退换货卡片
3. 点「扫地机器人不开机了」→ 按提示回复"不亮"→"仍不亮" → 排查树走到维修结论
4. 输入"转人工投诉" → 自动建高优先级工单
5. 点「结束会话并评价」打分
6. 打开管理控制台 → 工单看板拖状态 / 退换货审批
7. 学习审核 → 「开始分析历史会话」→ 审核一条候选入库 → 回聊天页问同一问题，AI 用新知识回答（**学习闭环**）
8. 知识库页 → 上传一份产品手册（PDF/Word/TXT/MD）→ 回聊天页问手册里的细节（如"尘盒容量多大"），AI 引用手册原文回答并标注出处（**手册投喂**）；点「消化为FAQ候选」让 LLM 提炼常见问答进审核队列
9. 数据分析页查看热点问题与满意度变化

> 手册注意事项：扫描版（纯图片）PDF 无法提取文字；复杂表格解析质量有限；旧版 .doc 需另存为 .docx。

## Docker 部署

```bash
docker compose up -d --build
# 环境变量通过宿主机环境或 .env（compose 变量替换）传入
```

数据（SQLite + 上传文件）持久化在 `./data` 卷。移动端：客户聊天页已做响应式适配（管理端以桌面为主）。

> 说明：Docker 镜像未在本机构建验证（构建耗时较长），Dockerfile/compose 按标准写法提供；生产上量还需把 SQLite 换为 PostgreSQL（涉及 FTS5 检索层替换为 pg 方案），当前定位为单机/小规模试用。

## 常用配置（.env）

| 变量 | 默认 | 说明 |
|---|---|---|
| `ANTHROPIC_API_KEY` | 空 | Anthropic 官方 API Key（优先级最高） |
| `OPENAI_API_KEY` | 空 | OpenAI 兼容接口 Key（中转站/国产模型） |
| `OPENAI_BASE_URL` | 空 | OpenAI 兼容接口地址，如 https://api.silra.cn/v1 |
| `MODEL` | 自动 | Anthropic→claude-opus-4-8；OpenAI兼容→qwen3-max |
| `MOCK_MODE` | auto | auto=无Key自动演示 / true=强制演示 / false=强制真实 |
| `DB_PATH` | data/app.db | SQLite 路径（删除该文件可重置演示数据） |
| `PORT` | 8000 | 服务端口（被占用时自动换空闲端口） |
